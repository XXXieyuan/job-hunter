### WBS 2: Rewrite APSJobsScraper with curl_cffi
**What**: Replace Playwright scrape() method with curl_cffi + Aura API
**Where**: scrapers/scrapers/apsjobs_scraper.py
**Input**: APSJOBS_REWRITE.md §Tech Details
**Steps**:
  1. Keep `extract_jobs_from_aura_response()`, `_clean()`, `parse_salary_from_string()`, `build_url()`
  2. Rewrite `scrape()` method:
     a. Create curl_cffi session with `impersonate="chrome120"`
     b. GET main page → extract `fwuid` via regex: `re.search(r'"fwuid"\s*:\s*"([A-Za-z0-9_]+)"', r.text).group(1)`
     c. Extract `app_id` via regex: `re.search(r'"APPLICATION@markup://siteforce:communityApp"\s*:\s*"([^"]+)"', r.text).group(1)`
     d. Loop over pages (offset = 0, 24, 48, ...)
     e. For each page: POST to `https://www.apsjobs.gov.au/s/sfsites/aura?r=5&aura.ApexAction.execute=1`
        - Headers: `Content-Type: application/x-www-form-urlencoded; charset=UTF-8`
        - Form data: `message`, `aura.context`, `aura.pageURI`, `aura.token=null`
        - `aura.context`: `{"mode":"PROD","fwuid":"<fwuid>","app":"siteforce:communityApp","loaded":{"APPLICATION@markup://siteforce:communityApp":"<app_id>"},"dn":[],"globals":{},"uad":True}`
        - `message` actions params: filter JSON with `searchString`, `offset`, `offsetIsLimit:False`, `cacheable:False`
     f. Parse JSON response → call `extract_jobs_from_aura_response()`
     g. Stop when `jobListingCount == 0` or offset exceeds known total
  3. Remove `playwright` import and all chromium/browser/page code
  4. Keep SSE event emitting (sse_emit) for progress feedback
  5. Handle errors gracefully with try/except + fallback message
**Output**: Updated apsjobs_scraper.py
**Verify**: `python scrapers/run_scraper.py --source apsjobs --keywords "data" --max_pages 2` → jobs printed
**Depends on**: WBS 1
